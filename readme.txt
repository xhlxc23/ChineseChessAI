================================================================================
                   中国象棋 AI 引擎 (Chinese Chess AI Engine)
================================================================================

一、项目简介
--------------------------------------------------------------------------------
本项目是一个基于 C++ 的中国象棋对弈引擎，适用于 Botzone 等在线对弈平台。
引擎采用 Alpha-Beta 剪枝搜索 + 静态评估的方式决策，具备开局库、置换表、
静态搜索、时间管理等功能，棋力达到较高水平。

二、编译与运行
--------------------------------------------------------------------------------
编译器要求：支持 C++17（如 g++-7 或更高版本）

编译命令（Botzone 平台）：
  g++-7 -D_BOTZONE_ONLINE -D_GLIBCXX_USE_CXX11_ABI=0 -O2 -Wall -std=c++1z \
  chineseChess2.cpp -ljson -lpthread -o ai

本地编译测试：
  g++ -O2 -Wall -std=c++17 chineseChess2.cpp -o ai

三、输入输出格式
--------------------------------------------------------------------------------
引擎通过标准输入（stdin）接收 JSON 格式的棋局信息，通过标准输出（stdout）
返回走法。

输入示例：
  {
    "requests": [{"source": "-1", "target": "-1"}],
    "responses": []
  }

输出示例：
  {"response": {"source": "h2", "target": "e2"}}

坐标格式：列用小写字母 a-i 表示（从左到右），行用数字 0-9 表示（从上到下）。
例如 "a0" 表示左上角，"i9" 表示右下角。

--------------------------------------------------------------------------------

  帅  K             将  k
  士  A             士  a
  象  B             象  b
  马  N             马  n
  车  R             车  r
  炮  C             炮  c
  兵  P             卒  p

五、核心算法
--------------------------------------------------------------------------------

1. 搜索算法
   - 主搜索：Alpha-Beta 剪枝（带置换表）
   - 静态搜索（Quiescence Search）：在叶子节点继续搜索吃子走法，避免
     水平线效应
   - 空着剪枝（Null Move Pruning）：在非将军局面下跳过己方走棋，加速搜索
   - 将军延伸（Check Extension）：对将军局面加深搜索

2. 置换表（Transposition Table）
   - 大小：2^20 条目（约 100 万）
   - 使用 Zobrist 哈希作为键值
   - 支持 EXACT / ALPHA / BETA 三种节点类型
   - 带 age 机制的替换策略

3. 走法排序（Move Ordering）
   - PV 走法优先（来自置换表）
   - 吃子走法按 MVV-LVA 排序
   - 杀手走法（Killer Moves，每层保留 3 个）
   - 历史启发（History Heuristic）
   - 将军走法加分
   - 靠近对方将帅的走法加分

4. 评估函数
   - 子力价值：帅 100000 / 车 600 / 炮 300 / 马 270 / 象 120 / 士 120 / 兵 70
   - 子力位置表（PST）：车、马、炮、兵各有独立的位置分值表
   - 子力安全：被攻击惩罚 / 被保护奖励
   - 将帅安全：被将军惩罚、活动空间评估
   - 士象完整性：缺士缺象扣分
   - 过河兵奖励
   - 马腿阻塞惩罚
   - 车炮机动性评估
   - 残局特殊评估（无大子时兵的价值提升）

六、规则处理
--------------------------------------------------------------------------------

1. 长将检测
   - 检测连续将军次数，防止长将犯规
   - 同一局面重复将军 3 次或连续将军 8 次将被判定为长将
   - 在走法生成阶段过滤长将走法

2. 来回走棋禁止
   - 检测走法是否为上一手走法的逆走法（同子从 A→B 再 B→A）
   - 同时追踪引擎自身上一手走法，防止隔回合的来回走
   - 优先选择非逆向走法，仅在无其他选择时允许逆向走法

3. 将帅对面检测
   - 走法合法性检查中包含将帅对面判定
   - 评估函数中检测将帅对面直接判胜负

七、时间管理
--------------------------------------------------------------------------------
   - 基础时限 650ms，根据走法数量和局面阶段动态调整
   - 迭代加深过程中每搜索 128 个节点检查一次时间
   - 剩余时间不足 100ms 时提前终止搜索
   - 绝对上限 850ms，确保不超过平台限制

八、开局库
--------------------------------------------------------------------------------
  内置小型开局库，覆盖常见开局走法：
  - 中炮对屏风马（h2e2 h9g7 h0g2 i9h9 i0h0 b9c7）
  - 中炮对反宫马
  - 仙人指路（g3g4）
  - 飞相局（c0e2）
  - 起马局（b0c2）

十、关键函数说明
--------------------------------------------------------------------------------
  main()                  -- 入口，初始化 Zobrist → 读输入 → 调用 AI
  getInputBotzone()       -- 解析 Botzone JSON 输入，恢复棋局状态
  ai()                    -- 迭代加深搜索主循环，选择最佳走法并输出
  minmax()                -- Alpha-Beta 搜索（带置换表、空着剪枝）
  quiescence()            -- 静态搜索
  evaluate()              -- 局面评估
  generate()              -- 走法生成
  is_legal_move()         -- 走法合法性检查
  attacked_by()           -- 判断某位置是否被某方攻击
  is_reverse_move()       -- 判断走法是否为来回走
  move_causes_long_check()-- 判断走法是否导致长将
  make_move() / unmake_move() -- 执行/撤销走法（更新 Zobrist 哈希）
  query_opening_book()    -- 开局库查询
